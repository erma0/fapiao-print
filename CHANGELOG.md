# 📋 更新日志

## v1.2.0 — 画质优化、OFD支持、金额识别、排版增强

### 🆕 新功能

- **OFD 格式支持**：支持打开 OFD（电子发票标准格式），自动提取嵌入图片
- **OCR 金额自动识别**：基于 WinRT OCR 引擎，零额外依赖，自动识别发票金额
  - 支持「价税合计（小写）」「价税合计¥」「合计」「金额」等关键字段
  - 图片发票加载后自动 OCR 识别
  - 扫描版 PDF 文字提取失败后 OCR 兜底
- **金额统计**：文件列表底部实时显示已选发票金额汇总
- **发票查验**：一键跳转国家税务总局发票查验平台
- **版面布局增强**：
  - 9 个预设布局（1×1 到 3×3）+ 自定义行列（1-10行 × 1-10列）
  - 工具栏快捷布局按钮，一键切换
  - 自动方向检测（行>列→纵向，否则横向）
- **缩放功能增强**：
  - Ctrl+滚轮无感缩放（聚焦指针处放大缩小）
  - 双击重置自适应
  - Ctrl++/-/0 快捷键
  - 自定义缩放组件（5% 步进，10%-500% 范围）
- **自适应 DPI 渲染**：小 PDF 页面（火车票等）自动提升渲染 DPI，确保打印清晰度
- **深色模式**：完整的深色主题支持，CSS 变量覆盖，localStorage 持久化
- **设置面板扩展**：PDF 自动打开、打印模式、主题切换、导出/恢复默认

### 🔧 改进

- **DPI 常量统一**：前端 `PDF_RENDER_DPI` 和 Rust 端 `RENDER_DPI` 统一为 300，默认值使用常量而非硬编码
- **Per-slot 边距**：每张发票在各自 slot 内有独立边距，1×2/2×2 布局下效果更精确
- **渲染结果增加 DPI 信息**：`RenderedPage` 新增 `render_dpi` 字段，前端记录实际渲染 DPI
- **WinRT 渲染自适应**：Rust 端根据页面尺寸自动调整渲染 DPI（最长边 ≥ 3508px）

### 🐛 修复

- **金额提取 — CID 字体 PDF 拼接缺陷**（严重）：`.join(' ')` → `.join('')`，修复 CID 编码 PDF 每个汉字独立项导致空格破坏正则匹配的问题
- **OCR 只识别第一页就中断**（中等）：移除三处 `break`，扫描版多页 PDF 全页 OCR
- **金额正则第3条过于严格**：`/价税合计\s+(\d+)/` → `/价税合计\s*([\d,，]+\.?\d*)/`，兼容无空格和有逗号场景
- **CJK 空格归一化**：OCR 文本和 CID 字体 PDF 中字间空格自动消除，提高金额匹配率
- **PDF.js 渲染 DPI 不足**：`scale: 3.0`（216 DPI）→ `targetDpi / 72`（≥300 DPI），确保 fallback 路径画质

## v1.1.0 — WinRT PDF渲染修复，中文发票完美显示

### 🆕 新功能

- **WinRT 原生 PDF 渲染**：使用 `Windows.Data.Pdf` API，正确渲染引用系统字体的中文 PDF
- **PDF.js 回退**：WinRT 失败时自动回退到 PDF.js 渲染
- **PDF.js CMap 配置**：中文 CID 编码字体正常渲染

### 🔧 改进

- **拖放文件**：Rust 端 `on_window_event` 监听 `DragDrop` → `window.eval()` 调用前端函数
- **打印功能**：`ShellExecuteW` 替代 `cmd` 调用，隐藏命令行窗口
